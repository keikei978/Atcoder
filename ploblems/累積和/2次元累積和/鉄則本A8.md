# 鉄則 A08 - Two Dimensional Sum（二次元累積和）

> ひとことで言うと: グリッドの「長方形領域の和」を何回も聞かれる問題。番兵込みの `(H+1)×(W+1)` テーブルを前計算し、各クエリを**包除（足す引く引く足す）の4項**で `O(1)` 回答。前計算 `O(HW)`、クエリ `O(1)`。

- 問題リンク: https://atcoder.jp/contests/tessoku-book/tasks/tessoku_book_h
- 難易度: 鉄則本 A08 / 1000点
- タグ: 二次元累積和, 累積和, 包除原理, 番兵

---

## 問題概要

H×W のグリッド。各マス (i,j) に整数 X が書かれている。

Q 個のクエリ「左上 (A,B)・右下 (C,D) の長方形領域の総和は？」に答える。

- `H,W ≤ 1500`、`Q ≤ 10^5`、座標は **1-indexed** で `1 ≤ A ≤ C ≤ H`

---

## やってはいけない方針（なぜ落ちるか）

各クエリで長方形を毎回ループして足すと：

- クエリ数: Q ≈ 10^5
- 1クエリの領域: 最悪 H×W ≈ 2.25×10^6

→ `O(Q×HW) ≈ 2×10^11`。完全に **TLE**。

「合計 × 何回も × そのままだと間に合わない」= 累積和の出番。今回はそれが2次元になっただけ。

---

## 核心：番兵込みテーブル + 包除の4項

### なぜ番兵（0行・0列）が要るか
累積和を**元の配列に上書き**で作ると、左端・上端を含む区間を計算するとき引く相手の添字が `-1` になって壊れる。
1次元で `S[0]=0` を置いたのと同じ発想で、2次元では **0行目・0列目をぜんぶ0** にした `(H+1)×(W+1)` テーブルを作る。
これで `x1-1` が 0 になっても番兵の0が入るので、端のクエリ（例: 左上から始まる領域）が自動で安全になる。

### テーブルの作り方（左上からの累積）
`S[i][j]` = 左上 (1,1) から (i,j) までの矩形の和。

```
S[i][j] = grid[i-1][j-1] + S[i-1][j] + S[i][j-1] - S[i-1][j-1]
                自分          上          左         左上を引く
```

「上」と「左」を足すと、左上の領域を**二重に**足してしまう。だから左上を1回引いて相殺する（包除）。

### クエリの式（包除の4項）
矩形 [x1..x2, y1..y2]（1-indexed, 両端含む）の和：

```
S[x2][y2] − S[x1-1][y2] − S[x2][y1-1] + S[x1-1][y1-1]
  全体        上のはみ出し   左のはみ出し   引きすぎた左上を戻す
```

上と左を引くと、左上の角を**二重に**引いてしまう。だから最後に1回足し戻す。
構築もクエリも「二重カウントを1項で打ち消す」同じ構造。

---

## 解法の手順

1. グリッドを読む
2. 番兵込み `(H+1)×(W+1)` のテーブル S を全0で用意
3. 左上から順に `自分+上+左−左上` で埋める（前計算 `O(HW)`）
4. 各クエリは4項の包除で `O(1)` 回答
5. 入力が多い（Q=10^5）ので `sys.stdin.readline` ＋ 一括出力で高速化

---

## 実装（Python）

<details>
<summary>コード</summary>

```python
import sys
input = sys.stdin.readline

H, W = map(int, input().split())
grid = [list(map(int, input().split())) for _ in range(H)]

# H+1, W+1 の二次元累積和テーブル（0行・0列は番兵で0）
S = [[0] * (W + 1) for _ in range(H + 1)]

# 左上から順に埋める
for i in range(1, H + 1):
    for j in range(1, W + 1):
        S[i][j] = grid[i-1][j-1] + S[i-1][j] + S[i][j-1] - S[i-1][j-1]
        #            自分            上          左          左上を引く（重複相殺）

def rect_sum(x1, y1, x2, y2):   # 1-indexed, 両端含む
    return S[x2][y2] - S[x1-1][y2] - S[x2][y1-1] + S[x1-1][y1-1]

A = int(input())
out = []
for _ in range(A):
    x1, y1, x2, y2 = map(int, input().split())
    out.append(rect_sum(x1, y1, x2, y2))

print("\n".join(map(str, out)))
```

</details>

### 動作確認
入力例1 → 出力 `25` / `56`（AC確認済み）。

## ハマりポイント / 学び

- **番兵を省くと端のクエリで詰む**。元配列に上書きで累積和を作ると `x1-1 = -1` で破綻する。`(H+1)×(W+1)` の0スタートが定石。← 今回詰まったのがまさにここ（`1 1 1 1` 的な端の入力）。
- 構築の `−左上` とクエリの `+左上` は、どちらも「二重カウントの相殺」。符号が逆なだけで発想は同じ。
- 1次元累積和の `S[0]=0` 番兵が、そのまま2次元の「0行・0列」に拡張されている。1次元を理解していれば地続き。
- Q が大きいので `input = sys.stdin.readline` と一括 `print` を忘れない（生の `input()` だと TLE 危険）。

## 計算量

| 方針 | 前計算 | 1クエリ | 全体 | 判定 |
|---|---|---|---|---|
| 各クエリで領域を愚直に足す | なし | O(HW) | O(Q×HW) ≈ 2×10^11 | ❌ TLE |
| 二次元累積和 | O(HW) | O(1) | O(HW + Q) ≈ 2×10^6 | ✅ AC |
