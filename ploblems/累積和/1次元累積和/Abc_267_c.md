# ABC267 C - Index × A(Continuous ver.)

> ひとことで言うと: 「長さ M の窓を1個ずつずらしながらスコアを更新」する問題。窓ずらしの**差分が区間和**になるので累積和で `O(1)` 更新。全体 `O(N)`。

- 問題リンク: https://atcoder.jp/contests/abc267/tasks/abc267_c
- 難易度: ABC C問題 / 300点
- タグ: 累積和, スライド窓, 差分更新

---

## 問題概要

長さ N の整数列 A から、長さ M の**連続部分列** B を選ぶ。

そのスコア `Σ i×B_i = 1×B_1 + 2×B_2 + … + M×B_M` の最大値を求める。

- 連続部分列なので「飛ばし選び」はできない（窓を選ぶだけ）
- `1 ≤ M ≤ N ≤ 2×10^5`、`A_i` は負もありうる

---

## やってはいけない方針（なぜ落ちるか）

各窓のスコアを毎回イチから計算すると：

- 窓の位置: 約 N 通り
- 1窓のスコア計算: M 回

→ `O(N×M) ≈ 4×10^10`。2秒では間に合わず **TLE**。

「全探索（BF）で行けるわけない」が正しい直感。

---

## 核心アイデア：窓をずらすと「前の窓の中身合計」を引くだけ

窓 `[5, 4]`（M=2）のスコア = 1×5 + 2×4 = 13
窓 `[4, -1]` のスコア = 1×4 + 2×(-1) = 2

この2つを見比べる。注目は残る要素 **4**。

- 前: 4 に重み **2**（2×4）
- 次: 4 に重み **1**（1×4）

→ 残る要素の重みが **1ずつ減る**。さらに左端 `5` が抜けて、右端に `-1` が重み M で入る。

「重みが1ずつ減る」＋「左端が抜ける」をまとめると、結局 **前の窓の中身合計をまるごと引く** だけになる。

```
新スコア = 前スコア
         − (前の窓の中身合計)      ← 残る要素の重み −1 ＋ 左端が抜ける分
         + M × (新しく入る右端)
```

確認: 13 − (5+4) + 2×(-1) = 13 − 9 − 2 = 2 ✓（[4,-1] のスコア2と一致）

---

## なぜ累積和が要るか

漸化式に出てくる「前の窓の中身合計」を毎回ループで足すと、また `O(N×M)` に逆戻り。

そこで**区間和を累積和で O(1)** にする。「区間和を何回も聞かれる」＝累積和の出番。

```
S[i] = A[0] + … + A[i-1]   （S[0]=0 の番兵込み）
A[l..r] の合計 = S[r+1] - S[l]
```

これで漸化式1ステップが O(1) → 全体 O(N) で AC。

---

## 解法の手順

1. 累積和 S を作る（区間和を O(1) で出すため）
2. 一番左の窓 `[0, M-1]` のスコアを**愚直に1回だけ**計算（スタート地点。前がないので漸化式が使えない）
3. 窓を右に1つずつずらし、漸化式で更新しながら最大値を取る

全体 `O(N)`。

---

## 実装（Python）

<details>
<summary>コード</summary>

```python
N, M = map(int, input().split())
A = list(map(int, input().split()))

# 累積和: S[i] = A[0]+...+A[i-1]
S = [0] * (N + 1)
for i in range(N):
    S[i+1] = S[i] + A[i]

def window_sum(l, r):   # A[l..r] の合計（両端含む）
    return S[r+1] - S[l]

# 最初の窓 [0, M-1] のスコアを愚直に計算（スタート地点）
score = 0
for i in range(M):
    score += (i+1) * A[i]

ans = score
# 窓を右に1ずつ: 前のscore − 前の窓の合計 + M×新しい右端
for l in range(1, N - M + 1):
    score = score - window_sum(l-1, l+M-2) + M * A[l+M-1]
    ans = max(ans, score)

print(ans)
```

</details>

### 添字メモ
- `l` = 新しい窓の左端
- 前の窓 = `A[l-1 .. l+M-2]` → `window_sum(l-1, l+M-2)`
- 新しく入る右端 = `A[l+M-1]`、重み M
- ループ範囲 `range(1, N-M+1)` は窓がはみ出さない左端の範囲

## ハマりポイント / 学び

- 初見でこの差分更新を思いつくのはキツい。**スライド窓＋差分更新**は典型パターンなので、一度やって体に入れる系。
- 「窓を1個ずらしたときの差分を式で書き出す」→ それが区間和になる、という流れが定石。
- 2次元累積和ではない。1次元の累積和1本で足りる。
- スタートの1窓だけは漸化式が使えないので愚直計算、という構成に注意。

## 補足：累積和の「一筋縄じゃいかない」発展パターン

今回の重み付きみたいに、素直な区間和から一歩ひねった系統まとめ。緑色目指す段階で押さえたいのは ★ 印。

### 重み付き・係数付き系（今回の仲間）
- **`Σ i×A_i` 型**: インデックスを重みにする。窓ずらしで差分が区間和になる（今回）。
- **重み付き累積和2本持ち**: `Σ A_i` と `Σ i×A_i` の2つを用意し、任意区間の重み付き和を O(1)。窓固定じゃなく任意区間版だとこっち。

### 差分・いもす法系
- ★ **いもす法**: 「区間に一律 +x」を何回もやってから最終状態を見る。差分配列を作って最後に累積和。区間加算クエリの定番。
- **二次元いもす**: グリッドに長方形加算。

### 多次元
- ★ **二次元累積和**: グリッドの長方形領域の和を O(1)。包除（足す引く足す引く）が入る。

### 条件カウント系（累積和＋データ構造）
- ★ **和がちょうど K の連続部分列の個数**: `S[r]-S[l]=K` を `S[l]=S[r]-K` の検索に変える。defaultdict と併用。
- **和が K の倍数の区間**: 累積和を mod K で分類してカウント。
- **和が K 以上の最短区間**: 累積和＋二分探索、または尺取り。

### 合わせ技
- ★ **累積和 × 二分探索**: 単調性を作って境界を探す。「平均が X 以上の区間が存在するか」系。
- **累積和 × 累積min**: `S[r] - min(S[0..r-1])` で最大区間和（実質 Kadane）。
- **XOR累積和**: `prefix_xor` で区間XORを O(1)。「区間XORが0の部分列」系。
- **GCD/積の累積**: 和以外の演算。積は0・負で注意。

### 近縁（累積和の親戚）
- **尺取り法 (two pointers)**: 和の単調性があるとき累積和の代わりに使える。
- **スライド最小値/最大値（単調デック）**: 固定幅窓の min/max を O(N)。今回の「窓ずらし」の親戚。

> 覚え方: 「窓ずらし → 差分が区間和 → 累積和」のセットでパターン認識しておくと、似た問題で気づける。
