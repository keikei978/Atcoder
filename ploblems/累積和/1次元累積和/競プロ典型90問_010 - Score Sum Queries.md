# 典型90 010 - Score Sum Queries（★2）

> 躓き: S1/S2 と添字の対応で混乱。1組はS1・2組はS2で揃える。区間和 = S[R] - S[L-1]、番兵 S[0]=0 を忘れない。

- 問題: https://atcoder.jp/contests/typical90/tasks/typical90_j
- アルゴリズム: 累積和（prefix sum）
- 難易度: ★2
- 解いた日: 2026-06-18

## 何の問題か
N人の生徒が1組か2組に属し、それぞれ点数を持つ。クエリ「学籍番号 L〜R の中で、1組の合計点・2組の合計点」が Q 個飛んでくる。各クエリに答える。

## 学んだこと
はじめて累積和を知った。区間の合計を何回も聞かれるとき、毎回足し直すと O(N×Q) でTLE。
前計算で「先頭からi番目までの合計」の表を作っておけば、区間和 = S[R] - S[L-1] の引き算1回で出せる。全体 O(N+Q) に落ちる。

組ごとに別々の累積和（S1, S2）を作るのがポイント。
- 1組の生徒のとき → S1 に点数を足す、S2 は前の値をコピー（=0を足したのと同じ）
- 2組の生徒のとき → その逆
こうすると「別の組の生徒は値0」になり、区間で切っても混ざらない。

## つまずきポイント
- S1/S2 のどっちに足すか、添字を何度も逆にした。「1組=S1、2組=S2」で固定する。
- クエリで A=S1[R]-S1[L-1]、B=S2[R]-S2[L-1] と、各組で同じ配列を引く。S2を混ぜない。
- ループは range(1, N+1)。S[0]=0 は番兵として残す。

## dpではない（メモ）
累積和は `S[i] = S[i-1] + a[i]` という漸化式の形を持つので一見DPっぽい。
でも「複数の選択肢から最適を選ぶ（max/min・場合分け）」がなく、ただ足し上げるだけの一本道なので、厳密にはDPとは呼ばない。前計算テクニックの一種。

## 計算量
- 前計算: O(N)
- クエリ: O(Q)（各 O(1)）
- 合計: O(N + Q) ≒ 20万回。愚直の O(N×Q) ≒ 100億回 から激減。

## コード
<details>
<summary>提出コード（Python）</summary>

```python
N = int(input())
S1 = [0] * (N + 1)
S2 = [0] * (N + 1)
for n in range(1, N + 1):
    C, P = map(int, input().split())
    if C == 1:
        S1[n] = S1[n-1] + P
        S2[n] = S2[n-1]
    else:
        S1[n] = S1[n-1]
        S2[n] = S2[n-1] + P

Q = int(input())
for q in range(Q):
    L, R = map(int, input().split())
    A = S1[R] - S1[L-1]
    B = S2[R] - S2[L-1]
    print(f"{A} {B}")
```
