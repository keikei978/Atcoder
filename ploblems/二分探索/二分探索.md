# 二分探索 (Binary Search)

ソート済みの範囲に対して、毎回半分に絞り込みながら目的を探すアルゴリズム。
計算量は `O(log N)`。範囲が 10^9 でも約 30 回のループで終わる。

---

## 2つの型を区別する

二分探索には大きく2タイプある。**「何を探しているか」で型が変わる**のが一番のキモ。

| 型 | 探すもの | 終了条件 | 例 |
|---|---|---|---|
| 値探索型 | ちょうど一致する値（存在保証あり） | `==` で即 return | A11 配列内の要素を探す |
| 境界探索型 | 条件を満たす最小（最大）の点 | 範囲が1点に収束 | A12 条件を満たす最小の時刻 |

ピッタリ一致する点が**存在するとは限らない**問題では、値探索型は使えない。
境界を探す。

---

## 値探索型（A11 - Binary Search 1）

配列に X が必ず存在する → 一致した瞬間に終われる。

```python
def 二分探索(A, a):
    left, right = 0, len(A) - 1
    while left <= right:
        mid = (left + right) // 2
        if A[mid] == a:
            return mid + 1   # 1-indexed に変換
        elif A[mid] < a:
            left = mid + 1   # 右半分へ
        else:
            right = mid - 1  # 左半分へ
    return -1
```

- 見るのは常に `mid = (left+right)//2`
- 外したら**範囲を半分捨てる**
- 一致点が存在するから `==` で抜けられる

`bisect` で楽する版:

```python
from bisect import bisect_left
print(bisect_left(A, X) + 1)
```

## 境界探索型（A12 - Printer）

求めたいのは「条件を満たす**最小の** t」。
ちょうど一致する点が存在しないことがあるので `==` で抜けてはいけない。

### なぜ `== ` で止めちゃダメか

プリンター問題で `f(t) = t秒までの総印刷枚数`、K=10 のとき:

```
t=5 → 9枚
t=6 → 12枚
```

`f(t)=10` ちょうどになる t は存在しない。9の次がいきなり12に飛ぶ。
`if X==K: return` で待つと永遠に一致せず -1 が返る。

### 探すのは「境界」

時刻を順に並べて「K枚に届いてるか」を書くと、単調増加だからきれいに2つに分かれる:

```
時刻:  1  2  3  4  5  6  7  8 ...
判定:  ×  ×  ×  ×  ×  ○  ○  ○ ...
                    ↑ここ！
                  ×と○の境目 = 答え
```

`×××…○○○…` の **×から○に切り替わる最初の○**、それが答え。
この `×|○` の境界線を絞り込むのが境界探索型。

```python
def 二分探索(A, K):
    left, right = 1, 10**9
    while left < right:
        mid = (left + right) // 2
        X = sum(mid // Ai for Ai in A)
        if X >= K:
            right = mid       # 届いてる → 答えはここか左、midは残す
        else:
            left = mid + 1    # 届いてない → 答えはもっと右
    return left
```

---

## 挟み撃ちのイメージ

`left` と `right` の2人で境界を両側から挟み撃ちする。

- `left` = 「答えはこれ以上のどこか」の下限
- `right` = 「答えはこれ以下のどこか」の上限
- 答えは必ず `left` と `right` の**間**にいる

mid を調べて:

- mid が ○（条件を満たす）→ 境界はここか左 → `right = mid`（midは候補に残す）
- mid が ×（満たさない）→ 境界はもっと右 → `left = mid + 1`（midは捨てる）

2人がジリジリ近づいて、ピッタリ重なった（`left == right`）ら終わり。
重なった一点が境界 = 答え。

---

## コードと処理の対応

```python
def 二分探索(A, K):
    left, right = 1, 10**9          # ① 挟み撃ちの2人を配置
    while left < right:             # ② 幅がある間ループ（幅0で終了）
        mid = (left + right) // 2   # ③ 真ん中を調べる
        X = sum(mid // Ai for Ai in A)  # ④ midが○か×か計算
        if X >= K:                  # ⑤ ○/×の判定
            right = mid             # ⑥ ○ → rightを引き寄せる
        else:
            left = mid + 1          # ⑦ × → leftを引き寄せる
    return left                     # ⑧ 重なった点 = 答え
```

役割分担:

- **幅を縮める** → ⑥⑦（どっちに転んでも `right-left` が必ず減る）
- **幅ゼロで止める** → ②
- **どっちに縮めるか決める** → ⑤

③は調べる場所を選ぶだけ、④は判定材料を作るだけ。

---

## なぜ必ず1点に収束するか

各ステップで範囲が**必ず狭くなる**:

- `X>=K` → `right=mid`。`mid<right` だったので right が小さくなる
- `X<K` → `left=mid+1`。left が大きくなる

どっちに転んでも `right-left` が毎回減る → いつか `left==right` → ループ終了。

### 無限ループしない理由

`mid=(left+right)//2` は `left<right` のとき必ず `left <= mid < right`（切り捨て）。
だから `right=mid` で right は必ず1以上小さくなり、`left=mid+1` は必ず前進する。

---

## 実行トレース（入力例1: A=[1,2,3,4], K=10）

`f(t) = t//1 + t//2 + t//3 + t//4`

```
f(4)=4+2+1+1=8
f(5)=5+2+1+1=9
f(6)=6+3+2+1=12
```

`left=1, right=8` まで縮んだ状態から:

```
① left=1, right=8   mid=4  f(4)=8   8>=10? No  → left=5
② left=5, right=8   mid=6  f(6)=12  12>=10? Yes → right=6
③ left=5, right=6   mid=5  f(5)=9   9>=10? No  → left=6
④ left=6, right=6   → left<right が False → 終了
   return 6  ✓
```

ステップ③→④で left が5から6に上がり、right の6に追いついて合流。
これが「初めて12枚以上になる最小の時刻=6」を指している。
