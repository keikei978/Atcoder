# 尺取り法 (Two Pointers / しゃくとり法)

ソート済みの配列に対して、2本のポインタ（左端・右端）を**前にしか動かさない**ことで、
全ペアを見ずに `O(N)` で区間を探索するアルゴリズム。

シャクトリムシが体を伸ばしたり縮めたりしながら前に進む動きに似ているのが名前の由来。

---

## 発動条件

尺取り法が使えるのは、次の性質が成り立つとき。

> **ソート済み + 条件が単調** → 「左端を進めると、許せる右端も戻らない」

この「**戻らなくていい**」性質が肝。これがあるから2本のポインタが合計でも `2N` 回しか動かず、
`O(N)` で済む。

---

## 例題: A13 - Close Pairs

> ソート済みの N 個の整数から、差が K 以下のペアは何通りあるか。

`A = [11, 12, 16, 22, 27, 28, 31]`, `K = 10` で考える。

### 素朴な方法だと間に合わない

全ペアの差を見ると `N(N-1)/2` 通り。N=10万だと約50億回で TLE。

### ソート済みのおいしさ

`i` を左端に固定すると、差が K 以下なのは **ある地点まで連続して、そこから先は全部アウト**。

```
i=0(11):
  11と12 → 差1   OK
  11と16 → 差5   OK
  11と22 → 差11  アウト！ ここから先は全部アウト
```

22がダメなら、その先の27,28,31はもっと差が大きいので見るまでもなくアウト。
**これがソート済みだからできること。**

---

## イメージ: シャクトリムシ

2本のポインタ `i`（左足）と `r`（右足）を使う。ルールは2つだけ。

- **`i`（左足）**: ペアの左側。1歩ずつ右に進む
- **`r`（右足）**: 「差が K 以下で行ける一番右」まで伸ばす

そして超重要: **どっちの足も前にしか進まない。戻らない。**

動き:
1. 左足 `i` を1歩進める
2. 右足 `r` を、行けるとこまで前に伸ばす（差が K 以下の間ずっと前へ）
3. その時の「i と r の間の個数」を数に足す
4. また左足を進める…

---

## コマ送り

各コマで「`i` と `r` の間に何個あるか」を数えて足す。

```
添字:  0    1    2    3    4    5    6
値:   11   12   16   22   27   28   31
```

### コマ① i=0（左足=11）

11+10=21 までOK。16(差5)はOK、22(差11)はアウト。

```
       i              
       ↓              
      11   12   16 │ 22   27   28   31
       └────┴────┘ │
      差0   差1   差5│ 差11←アウト
                    r=2でストップ
```

→ ペアの数 = 2個（11-12, 11-16）

### コマ② i=1（左足=12）

左足を1歩進めた。右足は**戻さない**。12+10=22なので22(差10)はギリOK、伸ばせる。

```
            i              
            ↓              
      11   12   16   22 │ 27   28   31
            └────┴───┘ │
            差4   差10  │差15←アウト
                       r=3でストップ
```

→ ペアの数 = 2個（12-16, 12-22）

### コマ③ i=2（左足=16）

16+10=26、27は差11でアウト。右足はそのまま。

```
                 i         
                 ↓         
      11   12   16   22 │ 27   28   31
                 └────┘ │
                 差6    │差11←アウト
```

→ ペアの数 = 1個（16-22）

### コマ④ i=3（左足=22）

22+10=32 まで OK。27(差5),28(差6),31(差9) 全部OK。一気に伸ばす。

```
                      i                   
                      ↓                   
      11   12   16   22   27   28   31 │(終端)
                      └────┴────┴───┘ │
                                      r=6まで
```

→ ペアの数 = 3個（22-27, 22-28, 22-31）

### コマ⑤〜⑦

```
i=4(27): r=6のまま → 2個 (27-28, 27-31)
i=5(28): r=6のまま → 1個 (28-31)
i=6(31): 右にもう無い → 0個
```

### 合計

```
コマ①  i=0:  2個
コマ②  i=1:  2個
コマ③  i=2:  1個
コマ④  i=3:  3個
コマ⑤  i=4:  2個
コマ⑥  i=5:  1個
コマ⑦  i=6:  0個
        ─────────
         合計 11個  ✓
```

---

## なぜ速いか（一番のキモ）

右足 `r` の動きに注目。コマ①→⑦で:

```
r:  2 → 3 → 3 → 6 → 6 → 6 → 6
```

**一度も戻っていない。** ずっと右、または据え置き。

- 左足 `i` は全部で N 歩
- 右足 `r` も戻らないので最大 N 歩

合わせて約 `2N` 回 = **O(N)**。全ペア `N²/2` を見なくて済む。

理由: 左端の値が大きくなれば、許せる右端も右にしか動かない（より大きい基準なのに
より厳しくなるのは矛盾）。だから右足を戻す必要が一切ない。

---

## コード

```python
N, K = map(int, input().split())
A = list(map(int, input().split()))

r = 0
ans = 0
for i in range(N):
    # 右足を、差がK以下で行けるとこまで伸ばす（戻さない）
    while r < N and A[r] - A[i] <= K:
        r += 1
    # r は初めてアウトになった位置（or 終端）
    # i とペアを組めるのは i+1 .. r-1 の (r-1 - i) 個
    ans += (r - 1) - i

print(ans)
```

ポイント:

- `r` はループの**外**で持ち、i が進んでもリセットしない（これが尺取りの本体）
- `while` は「A[r] がまだ OK な限り r を前に進める」
- 止まった `r` は「初めてアウトな位置」なので、OK な右端は `r-1`。足すのは `(r-1) - i` 個

## 二分探索との関係

同じ問題が二分探索でも解ける（A13 は両方OK）。

| 手法 | 計算量 | 特徴 |
|---|---|---|
| 尺取り法 | O(N) | 区間が単調に動くとき。最速 |
| 二分探索 | O(N log N) | 各点で独立に境界を探す。考え方が素直 |

二分探索版（各 i について A[i]+K 以下の最大添字を探す）:

```python
from bisect import bisect_right
ans = 0
for i in range(N):
    j = bisect_right(A, A[i] + K) - 1
    ans += j - i
print(ans)
```

迷ったら二分探索で書いて、速度が足りない時だけ尺取りに直す、で実戦は回る。

## 尺取りが使える他の問題タイプ

「区間を伸び縮みさせて条件を保つ」系で発動する。

1. **和が条件を満たす連続区間**: 「和が K 以下になる連続区間の数 / 最長の長さ」。
   右端を伸ばして和が超えたら左端を縮める。**正の数限定**（負があると単調性が崩れる）。
2. **条件を満たす最短区間**: 「合計が S 以上になる最短の連続区間」。
3. **種類数の制約（可変窓）**: 「異なる要素が K 種類以下になる最長の連続区間」。
   文字列問題でも頻出。
4. **2配列のマージ的探索**: ソート済み2配列で「和が K 以下のペア数」。

---

## 頭に残す3点

1. **ソート済みだから「ある地点から先は全部アウト」が成立** → 右足を戻さなくていい
2. **左足も右足も絶対に戻らない** → 合計 2N 回 = O(N)
3. **各コマで区間内の個数を足す**

- [ ] 配列はソート済みか（前提が崩れると使えない）
- [ ] 条件は単調か（左端を進めたら右端も戻らないか）
- [ ] 和の問題なら、値はすべて正か（負があると尺取り不可）
- [ ] `r` をループ外で持ち、リセットしていないか
- [ ] 数える個数の定義（`r-1 - i` か `r - i` か）を入力例で検算したか
