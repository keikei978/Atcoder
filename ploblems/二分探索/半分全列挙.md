# 半分全列挙 (Meet in the Middle)

全探索すると間に合わない問題を、**対象を2つに分割して、それぞれを別々に全列挙**し、
あとで突き合わせることで計算量を劇的に下げるテクニック。

「真ん中で出会う (meet in the middle)」が名前の由来。
分割した両側を別々に列挙して、中間で照合する。

---

## 発動条件

> 全部を一度に探索すると指数 / 高次が爆発するが、**半分に割ると現実的なサイズに収まり、
> 片方を表（set や ソート配列）にして照合できる** とき。

典型は計算量が `O(M²)` や `O(2^M)` のものを、
半分にして `O(M)` × `O(M)` や `O(2^(M/2))` に落とすパターン。

---

## 例題: A14 - Four Boxes

> 4つの箱 A,B,C,D から1枚ずつ取り、合計が K になる組み合わせはあるか。

### 素朴な方法だと間に合わない

4重ループで全組み合わせ = `N⁴`。N=1000 なら `10¹²` で TLE（5秒でも無理）。

### 真ん中で割る

4つの箱を **前半 (A,B)** と **後半 (C,D)** に分ける。

- 前半: `a + b` の全組み合わせ → `N² = 100万` 通り
- 後半: `c + d` の全組み合わせ → `N² = 100万` 通り

合計が K になる = `(a+b) + (c+d) = K`
→ つまり **`a+b = K - (c+d)`** が成り立つペアがあるか。

### 照合のしかた

1. 前半 `a+b` を全部 **set** に入れる（100万個）
2. 後半 `c+d` それぞれについて、`K - (c+d)` が set にあるか O(1) で照会

```
   前半 A+B                    後半 C+D
 ┌──────────┐             ┌──────────┐
 │ a+b を全列挙 │             │ c+d を全列挙 │
 │  → set に格納│  ←─照合─→  │ K-(c+d) を   │
 │  (100万個)  │             │ set で引く   │
 └──────────┘             └──────────┘
        真ん中で出会う (meet in the middle)
```

### 計算量

- 前半の列挙: `O(N²)`
- 後半の照合: `O(N²)` × set参照 `O(1)` = `O(N²)`
- 合計 **`O(N²)`** = 100万程度。余裕で間に合う。

`N⁴ = 10¹²` → `N² = 10⁶` への激減がこのテクニックの威力。

---

## コード

```python
N, K = map(int, input().split())
A = list(map(int, input().split()))
B = list(map(int, input().split()))
C = list(map(int, input().split()))
D = list(map(int, input().split()))

# 前半 A+B の和を全部 set に入れる（N²通り）
AB = set()
for a in A:
    for b in B:
        AB.add(a + b)

# 後半 C+D それぞれについて、残り K-(c+d) が AB にあるか
found = False
for c in C:
    for d in D:
        if K - (c + d) in AB:
            found = True
            break
    if found:
        break

print("Yes" if found else "No")
```

ポイント:

- 前半を `set` にするのが肝。`x in set` は平均 `O(1)`
- `break` 二段は「見つかったら即終了」の枝刈り（無くても通るが速い）

---

## 照合方法の選択: set か 二分探索か

前半を表にして後半で引くとき、引き方は2通り。

| 方法 | 計算量 | 使う場面 |
|---|---|---|
| **set の `in`** | 平均 O(1) | ちょうど一致するか（存在判定）だけでいいとき |
| **ソート + 二分探索** | O(log N) | 「K以下の個数」「一番近い値」など境界が必要なとき |

A14 は「ちょうど K になるか」の存在判定なので **set が最速**。
鉄則本が「二分探索で」と書いているのは C++ 想定（std::set やソート+二分探索が定番）。
Python なら set 一択。

ただし「合計が K **以下**になる組は何通り？」のような**個数・境界**を聞く問題なら、
後半をソートして二分探索（bisect）が必要になる。

---

## 半分全列挙が使える他の問題タイプ

1. **部分集合の和（subset sum）**: N 個から部分集合を選んで和を S にできるか。
   `2^N` を `2^(N/2)` × `2^(N/2)` に分割。N=40 くらいまで対応可能。
2. **k個の配列から1つずつ選んで合計が条件を満たすか**: A14 の一般化。
   配列を前半グループ・後半グループに分ける。
3. **和が S 以下になる部分集合の数**: 後半をソートして二分探索で個数を数える。

共通する判断軸:

> 「全部やると `2^N` や `N^k` で爆発」かつ「半分に割れば表にして照合できる」
> → 半分全列挙を疑う。

---

## 二分探索・尺取りとの関係

今日つながった3つの関係を整理:

| テクニック | 何をする | 計算量の典型 |
|---|---|---|
| 二分探索 | ソート済みで値・境界を探す | O(log N) |
| 尺取り法 | 区間を単調に動かして数える | O(N) |
| 半分全列挙 | 2分割して片方を表にし照合 | 爆発 → 平方根的に減 |

半分全列挙の「照合」フェーズで、set（一致）や 二分探索（境界）を使う。
**半分全列挙は枠組み、set/二分探索はその中で使う道具**、という階層関係。

---

## 頭に残す3点

1. **全部やると爆発する → 真ん中で2分割**
2. **片方を表（set / ソート配列）にする**
3. **もう片方で「残り」を照合**（一致なら set、境界なら二分探索）
